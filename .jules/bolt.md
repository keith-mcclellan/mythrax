## 2024-06-12 - Replacing .clone().clear() with std::mem::replace in Text Parsing Pipelines
**Learning:** In Mythrax's intensive chunking logic (`split_by_chars` and `group_sub_chunks` in `src/vault/ingestion.rs`), replacing `string.clone()` followed by `string.clear()` with an immediate `std::mem::replace(&mut string, String::with_capacity(max_chars))` bypasses unnecessary O(N) memory copies. This is critical because `String::clear()` only zeros the length but leaves the buffer intact; however, `.clone()` reallocates a brand new buffer every time, leading to heavy heap fragmentation during large document ingestion. Overwriting the local buffer with `String::with_capacity` avoids reallocating on subsequent character pushes.
**Action:** When auditing tight loops for string building/chunking in Rust pipelines, proactively search for `.clone()` followed by `.clear()` or variable overwrites, and substitute them with direct moves using `std::mem::replace`.
