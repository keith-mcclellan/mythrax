## 2024-07-15 - SurrealDB Table Name String Interpolation Vulnerability
**Vulnerability:** A SQL injection vulnerability was found in `get_node_scope` where the query string was constructed using `format!("SELECT scope FROM {};", rec_id.table)`. `rec_id.table` comes from user input via `parse_record_id`, allowing a malicious user to manipulate the query table name or execute arbitrary queries.
**Learning:** String interpolation of table names in SurrealQL can lead to SQL injection if derived from untrusted sources, even if it looks like an indirect ID parse. Furthermore, `SELECT ... FROM table` queries the entire table, whereas `SELECT ... FROM $id` queries a specific record by binding the record ID directly.
**Prevention:** Use direct Record ID bindings (`$id`) when selecting a specific record, which avoids string interpolation and provides natural parameterized execution for SurrealDB. If dynamic table names must be used, they must be securely validated or parameterized via `type::table($param)`.
