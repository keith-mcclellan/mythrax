## 2024-07-14 - Fix SQL Injection via SurrealDB String Interpolation
**Vulnerability:** Found multiple instances of SQL injection vulnerability where dynamic table names were directly interpolated into SQL query strings using `format!()` in `surrealdb` queries (e.g., `let sql = format!("SELECT scope FROM {};", rec_id.table);`).
**Learning:** Using `format!()` for dynamic table names or binding record IDs into string queries allows for SQL injection if the table name or other parameters are somehow maliciously crafted. The `RecordId` structure handles escaping of the key part but the `table` name part could potentially be manipulated. The memory file states: "When using SurrealDB in Rust, parameterize dynamic table names using type::table($param) instead of string interpolation (e.g., format!()) to prevent SQL injection vulnerabilities. To select a specific record by ID, use SELECT ... FROM $id and bind the surrealdb::types::RecordId directly."
**Prevention:** Avoid `format!()` for dynamic parameters like table names or record IDs. Use `SELECT ... FROM type::table($tb)` with `.bind(("tb", table_name))` or `SELECT ... FROM $id` with `.bind(("id", rec_id))`.
