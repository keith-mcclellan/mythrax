## 2024-05-20 - SurrealDB String Interpolation Injection Risk with parse_record_id
**Vulnerability:** A SQL injection vulnerability was found in `get_node_scope` where the table name was interpolated directly into a SurrealDB query via `format!("SELECT scope FROM {};", rec_id.table)`.
**Learning:** Even when inputs are partially validated or parsed (e.g., through `parse_record_id`), interpolating strings directly into a query (like `.table`) allows an attacker to inject arbitrary commands or manipulate the target if the parsing logic doesn't strictly strip out malicious payloads or if the type itself is malleable. In this specific case, selecting a specific record should be done using `$id` directly.
**Prevention:** Never use string interpolation (e.g., `format!()`) to build queries for dynamic table names or records. Always use SurrealDB's parameter binding logic. For selecting a specific record, `SELECT ... FROM $id` bound to the `surrealdb::types::RecordId` provides strict isolation. For querying a table dynamically, use `type::table($param)`.
